# 协作者手册

## 可以贡献什么

- 往年试卷 → `exam/<课程名>/`
- 课后作业 → `homework/<课程名>/`
- 补充资料 → `supplement/<课程名>/`

试卷命名示例：`高等数学A（II）-25春-期末.tex`；作业命名示例：`第3次作业.tex`；资料可自行命名，电子资料（如宝藏网站、参考文献）可写入 `supplement/<课程名>/source.md`。

## 排版规则

- 每个 .tex 开头写 `\heading{课程名-说明}`，`question` 环境包题目，`solution` 环境包解答
- 配图用 `\pyfig[宽度]{路径}{说明}`，图放在对应课程的 `figures/` 文件夹

## 工作流程

```bash
# 1. 基于最新 master 创建分支
git checkout master
git pull --rebase
git checkout -b feat/添加xxx

# 2. 添加或修改文件后提交
git add -A
git commit -m "feat: 添加xxx"

# 3. 提交前检查改动（这一步不能省）
git fetch origin
git diff origin/master --name-status

# 4. 确认没有误删、误改后推送并创建 PR
git push origin feat/添加xxx
gh pr create --base master --title "feat: 添加xxx" --body "添加了xxx资料"
```

## 原则

- 只加新文件，不删不改旧文件（除非你 100% 确定）
- 只动自己的课程目录，别碰其他人的
- 看到上述 `git diff` 出现意外的 `D`（删除）或 `M`（修改），先 `git pull --rebase origin master` 同步
- 拿不准就来问，别自己直接删

## 代码规范

### 导言区（`preamble.tex`）提供的命令与环境

以下命令已在全局 `preamble.tex` 中定义，所有 `.tex` 文件**无需重复定义**，直接使用即可：

| 命令 | 用途 | 示例 |
|------|------|------|
| `\ee` | 自然对数的底 \(\mathrm{e}\) | `\ee^{\ii z}` |
| `\ii` | 虚数单位 \(\mathrm{i}\) | `\ii`, `\cos z + \ii \sin z` |
| `\dd` | 微分算符（由 `physics` 宏包提供） | `\dd{x}`, `\dd[2]{x}` |
| `\heading` | 作业/试卷标题 | `\heading{数学物理方法（上）-第1次作业}` |
| `question` | 题目环境（自动编号） | `\begin{question} ... \end{question}` |
| `solution` | 解答环境（仅当 `\ShowSolutionstrue` 时显示） | `\begin{solution} ... \end{solution}` |
| `\pyfig` | 插入 Python 生成的图片 | `\pyfig[宽度]{路径}{说明}` |
| `\Beta` | Beta 函数 \(\mathrm{B}\) | `\Beta(a,b)` |
| `\J` | Bessel 函数（第一类）\(\mathrm{J}\) | `\J_\nu(x)` |

### 排版规范

1. **枚举列表** — 题目中的子问题统一使用 `enumerate` 环境（不要手写 `1.` `2.` 或 `(1)` `(2)`）。解答中也同样使用 `enumerate`。
2. **枚举编号格式**（已在 `preamble.tex` 中全局设置）：
   - 第一层 `enumerate` 使用 `(a)(b)(c)` 编号；
   - 第二层嵌套使用 `(i)(ii)(iii)` 编号。
3. **解答结构** — `solution` 环境内的子问题编号应与题目保持一致，同样使用 `enumerate`。
4. **多种方法** — 同一题目给出多种解法时，方法名称用 `\textbf{}` 标出即可，下方内容不要缩进，不要使用 `enumerate` 嵌套。
5. **数学常数** — 使用全局命令 `\ee`、`\ii`，不要直接在公式中写 `e` 或 `i`。
6. **微分算符** — 使用 `\dd{x}` 代替 `dx`，统一为直体微分符号。
7. **括弧顺序** — `\end{solution}` 必须出现在 `\end{question}` **之前**（`solution` 嵌套在 `question` 内部）。

### 配图规范

1. 图片统一存放在对应课程目录下的 `figures/` 文件夹。
2. 命名规则：**`<tex文件名>-fig<NN>.<扩展名>`**，例如：
   - `第1次作业-fig01.png`
   - `第7次作业-fig01.png`
3. 插入方式：`\includegraphics[width=...]{figures/文件名}` 或 `\pyfig[宽度]{figures/文件名}{说明}`。
4. 无用的图片请及时清理，避免遗留未引用的文件。

### 提交信息规范

| 类型 | 描述 | 示例 |
| :--- | :--- | :--- |
| **`feat`** | **新功能** (feature) | `feat: 添加用户登录功能` |
| **`fix`** | **修复 Bug** | `fix: 修复订单金额计算错误` |
| **`docs`** | **文档**变更 | `docs: 更新API接口文档` |
| **`style`** | 代码**格式**调整（不影响功能，如空格、格式化） | `style: 格式化代码，调整缩进` |
| **`refactor`** | **代码重构**（既非新功能也非修Bug） | `refactor: 抽离通用接口逻辑` |
| **`perf`** | **性能优化** | `perf: 使用索引提升查询效率` |
| **`test`** | 增加或修改**测试**代码 | `test: 增加登录功能的单元测试` |
| **`chore`** | **构建工具**、**依赖管理**等杂项 | `chore: 升级eslint到v8.56` |
| **`ci`** | **持续集成**（CI）的配置和脚本更改 | `ci: 配置GitHub Actions工作流` |
| **`build`** | 影响**构建系统**或外部依赖的更改 | `build: 更新webpack配置` |
| **`revert`** | **回滚**之前的提交 | `revert: 回滚上一次的用户登录功能修改` |
