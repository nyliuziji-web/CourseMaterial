\begin{question}
当各边上的权值 \blank 时，BFS 算法可用来解决单源最短路径问题。
\begin{choiceoptions}[2]
  \item 均相等
  \item 均互不相等
  \item 不一定相等
\end{choiceoptions}
\begin{solution}
选 A。BFS 的层次扩展保证先到达的顶点经过的边数最少，因此它求的是"边数最少"的路径。若所有边权值均相等，则路径权值正比于边数，BFS 得到的就是单源最短路径；若边权不同，即使边数少也未必权值小，应改用 Dijkstra 等带权最短路算法。
\end{solution}
\end{question}

\begin{question}

\begin{question}

\begin{solution}
最佳 4 路归并树 WPL 为 $480$。4 路 Huffman 合并要求叶子数满足 $(n-1)\bmod(4-1)=0$；原有 13 个归并段已满足该条件，因此不需要补 0 长度虚段。每次取四个最短段合并：
\[
5,9,12,13\to39,\quad
14,16,17,18\to65,\quad
20,28,30,37\to115,\quad
39,42,65,115\to261.
\]
带权外部路径长度等于各次合并长度之和，故 $WPL=39+65+115+261=480$。
\end{solution}
\end{question}

\begin{question}
在一棵阶为 k 的红黑树中，内部结点最少有2k-1 个；从根到叶的简单路径长度的范围为 [k,2k]。
\begin{solution}
该说法按"黑高度为 $k$"理解时成立：红黑树中从某结点到任一外部叶子的黑结点数相同，且红结点不能有红孩子，因此黑高度为 $k$ 时，内部结点数至少出现在全黑的最瘦情形，至少为 $2^k-1$；任一路径上红结点数不超过黑结点数，所以实际高度至多约为 $2k$。若题面中的"阶为 $k$"是指普通高度，则"最少有 $2k-1$ 个内部结点"并不一定成立，表述应改成关于黑高度的命题。
\end{solution}
\end{question}

填空为：\texttt{\detokenize{i == j and length == 0}}；以及 \texttt{\detokenize{total += get_path_num_len(G, v, j, length - 1)}}。递归含义是“从当前顶点 $i$ 出发，还需要走 $length$ 条边到达 $j$ 的简单路径条数”。当 $length=0$ 时，只有当前顶点已经是 $j$ 才计为一条路径；否则应枚举 $i$ 的每个未访问邻接点 $v$，标记后递归求从 $v$ 到 $j$、剩余长度为 $length-1$ 的路径数，并累加到 \texttt{total}。返回前必须恢复 \texttt{visited[i]}，否则会影响其他分支的枚举。
\end{solution}

\begin{question}
伸展树是一种自平衡的BST树，它可以通过旋转来实现树结构的动态调整。伸展树结点的数据结构定义如下：
\begin{pycode}
class TreeNode:
    def __init__(self, data):
        self.data = data
        self.father = None
        self.left = None
        self.right = None
\end{pycode}
其成员函数包括：
1）\texttt{\detokenize{Delete(x)}}：删除以 $x$ 结点为根的子树；
2）\texttt{\detokenize{Splay(x, f)}}：将 $x$ 旋转为 $f$ 的子结点（$f$ 是 $x$ 的祖先）。特殊地，把 $x$ 旋转到根结点即 \texttt{\detokenize{Splay(x, None)}}。
请运用上述给定的成员函数，实现删除树rt中所有大于u小于v的结点（假设u, v是Splay树中的结点，且树中所有结点值不重复）。
函数原型为：\texttt{\detokenize{delete_between(root, u, v)}}。
解答：把u 结点旋转到根；把v 旋转为u 的右儿子；删除v 结点的左子树前面几句各3 分，最后一句1 分。如果思路、注释各1 分，不写则扣。
\begin{pycode}
def delete_between(root, u, v):
    Splay(u, None)
    Splay(v, u)
    Delete(v.left)
    v.left = None
    return u
\end{pycode}

\begin{solution}
先执行 \texttt{\detokenize{Splay(u, None)}}，使 $u$ 成为整棵树的根。由于 $u<v$，所有落在区间 $(u,v)$ 中的关键码都位于根 $u$ 的右子树内；再执行 \texttt{\detokenize{Splay(v, u)}}，只在 $u$ 的右子树中把 $v$ 伸展为 $u$ 的右孩子。此时 $v$ 的左子树恰好包含所有大于 $u$ 且小于 $v$ 的结点，而 $v$ 的右子树包含大于 $v$ 的结点。调用 \texttt{\detokenize{Delete(v.left)}} 释放该左子树并令 \texttt{\detokenize{v.left = None}}，即可删除开区间内全部元素，同时保留区间外元素的二叉搜索树次序。
\end{solution}
\end{question}
